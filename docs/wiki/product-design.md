# 产品设计

## 第一批用户

MVP 面向爸妈、爷爷奶奶，其中老人是核心用户。产品不应表现为网盘、NAS、全量备份工具或素材管理器，而应表现为打开就能看宝宝近况的家庭精选照片流。

MemoTree 的主心智是“家人每天打开看看宝宝近况”。全量照片备份由其他工具或人工流程承担，MemoTree 只承载家庭成员愿意共享给家人看的精选照片和视频。

## 核心优先级

1. 首屏看图：打开首页直接进入家庭精选近况时间线。
2. 上传可见但次要：保留明显上传入口，但不抢占浏览内容。
3. 私密访问：非家庭成员不可访问，移除成员后立即失去访问能力。
4. 原文件私有保存：下载入口后置，但后端必须从第一版保存原文件并避免长期公开 URL。

## 家庭空间

MVP 使用 `Family` 作为私密共享空间和权限边界，不引入宝宝、相册或文件夹层级。

- `Family` 表示一个家庭精选近况空间，不表示某一个宝宝。
- `User` 和 `Family` 是独立领域对象，通过 `FamilyMember` 形成多对多关系。
- `FamilyMember` 表示用户在某个家庭里的称呼、角色和状态。
- MVP 不引入 `Child` 模型，也不要求上传时选择具体宝宝。
- MVP UI 默认按单家庭体验设计，但领域模型允许一个用户加入多个家庭。

## 时间线

首页主时间线表达宝宝和家庭近况的成长顺序，优先按拍摄时间展示。

- 首页按 `captured_at` 倒序排列，并按月份、日期分组。
- 如果媒体缺失拍摄时间，则使用 `uploaded_at` 混入时间线。
- 上传时间排序后续可作为“最近添加”视图补充，不进入 MVP 主流程。
- 详情页可以展示拍摄时间、上传者和上传时间。

## 媒体资产

上传到 MemoTree 的内容属于家庭空间，上传者只是贡献者和审计信息。

- `MediaAsset` 表示时间线上的一个媒体条目，可以是 `photo`、`video` 或 `live_photo`。
- 一次批量上传会创建多个独立 `MediaAsset`，不引入帖子、相册、批次或动态分组。
- 时间线永远展示 `MediaAsset`，不会直接展示原始文件。
- `MediaOriginal` 保存用户上传的原文件，例如 HEIC、JPG、MOV 或 MP4。
- `MediaRendition` 保存给前端展示用的派生资源，例如 `thumbnail`、`display_image` 和 `display_video`。
- 原文件用于私有存储和后续授权下载预留，展示资源用于时间线和详情页渲染。
- iPhone 实况照片合并为一个 `live_photo`，包含图片原文件和视频原文件；MVP 默认展示静态图片，不把实况视频单独展示到时间线。
- `admin` 可以软删除家庭媒体；普通 `member` 不能删除媒体。
- 软删除后的媒体不出现在时间线和详情中，但原文件可等待后台或人工清理。

## 上传体验

MVP 使用同一套上传能力覆盖移动端和桌面端。移动端用于日常少量精选分享，桌面端可用于初始化批量上传。

- 上传入口支持从浏览器文件选择器多选照片和视频。
- 暂不做拖拽上传和拍照/录像直传入口。
- `UploadBatch` 表示一次可返回查看的上传任务。
- `UploadItem` 表示上传任务中的单个文件。
- `UploadBatch` 和 `UploadItem` 管理上传流程，不作为时间线浏览对象。
- 同一用户在同一家庭同一时间只能有一个 active 上传任务；再次点击上传时进入当前任务页。
- 上传者可以查看自己的上传任务，`admin` 可以查看家庭内所有上传任务。
- 浏览器正在上传原文件时，页面关闭可能中断；原文件上传完成后的展示资源处理可以由后台继续执行。
- 首页只展示 `status=active` 且 `rendition_status=ready` 的媒体；处理中和失败项只在上传任务页展示。
- 上传失败可以重试该文件；处理失败可以重试生成展示资源。
- 上传者或 `admin` 可以停止上传任务，已完成的媒体保留，未完成项取消。
- MVP 上传限制先采用保守值：单批最多 100 个文件，照片最大 50 MB，视频最大 1 GB，并通过配置调整。

## 成员与权限

MVP 只区分管理员和普通成员。

- `admin` 可以邀请成员、撤销未使用邀请、移除成员、管理家庭基础信息和成员称呼。
- `member` 可以查看时间线、查看媒体详情和上传照片/视频。
- 媒体访问只要求当前用户是 active member。
- 管理动作要求当前用户是 active admin。
- 移除成员只移除其家庭成员身份，不删除全局用户，也不删除其历史上传内容。
- 应避免家庭空间失去最后一个 active admin。

## 登录与邀请

MVP 采用持久化账号 + 密码登录，并通过家庭邀请控制首次加入。邀请负责判断用户是否有资格加入某个家庭，账号密码负责后续重复登录，浏览器会话只负责减少重复输入。

- 用户账号长期落库。
- 用户通过登录名和密码证明自己是同一个全局账号。
- MVP 的登录名可以优先使用手机号，但手机号在未接入短信验证码前只作为登录名，不作为强身份验证。
- 管理员创建一次性邀请。
- 一个邀请链接最多成功绑定一个账号。
- 用户通过邀请获得家庭成员权限。
- 被移除成员不能使用旧邀请重新加入，必须重新邀请。
- 用户忘记密码时，MVP 可以先由家庭管理员重置密码或重新邀请，短信/邮箱找回后置。

当前实现状态：

- 已支持账号注册、账号密码登录、退出登录和浏览器会话恢复。
- 已支持登录用户创建家庭，创建者自动成为家庭 `admin`。
- 已支持 `admin` 创建邀请、查看最近邀请、复制待使用邀请链接、撤销待使用邀请。
- 邀请状态包括 `pending`、`used`、`revoked` 和前端展示用的 `expired`。
- 为了让管理员刷新页面后仍能复制未使用邀请，MVP 当前会在数据库保存 `token_plaintext`；邀请被使用或撤销后会清空 token 原文。
- 普通 `member` 不能创建、查看或撤销邀请。
- 成员管理、移除成员、管理员转让和最后一个管理员保护还未实现。

## MVP Scope

### 必须做

- 家庭私密精选近况流，不做全量备份。
- `Family` 作为私密共享空间和权限边界。
- `User` 和 `Family` 多对多，通过 `FamilyMember` 连接。
- 仅 `admin / member` 两类角色，成员状态为 `active / removed`。
- 邀请门控注册，账号密码登录。
- admin 创建一次性邀请，成员通过邀请加入家庭。
- 首页使用按日期分组的紧凑媒体网格。
- 主时间线按 `captured_at ?? uploaded_at` 倒序展示。
- 默认时区使用 `Asia/Shanghai`，MVP 不提供时区设置。
- `MediaAsset / MediaOriginal / MediaRendition` 媒体模型。
- 支持 `photo / video / live_photo`。
- 私有保存原始照片和视频文件。
- 生成 `thumbnail / display_image / display_video` 展示资源。
- 实况照片合并为一个 `live_photo`，默认展示静态图。
- 移动端和桌面端文件选择批量上传。
- `UploadBatch / UploadItem` 可返回查看上传任务。
- 同一用户同一家庭最多一个 active 上传任务。
- 默认上传限制：100 文件/批，照片 50 MB，视频 1 GB，配置化。
- 上传失败可重试，处理失败可重试生成展示资源。
- 上传任务可停止，已完成媒体保留，未完成项取消。
- 首页只展示 `status=active` 且 `rendition_status=ready` 的媒体。
- 媒体详情页展示图片/视频、基础信息和信息面板。
- 详情页尽量支持上一张/下一张切换。
- admin 可以软删除媒体和修改拍摄时间。
- 媒体处理阶段提取 `captured_at` 入库，admin 可单张修正。

### 预留但不做

- 原文件下载：保存 `MediaOriginal`，不暴露 object key。
- 实况照片动态播放：可预留 `display_video`，MVP 不提供播放按钮。
- 最近添加视图：保留 `uploaded_at`，不作为 MVP 主入口。
- 多家庭 UI：模型支持多家庭，UI 只做简单选择。
- 时区设置：`Family.timezone` 默认 `Asia/Shanghai`。
- 上传任务历史清理：先保留，UI 默认展示最近几条。
- 批量下载：原文件模型预留，后续再定。
- 分片上传和断点续传：上传任务模型预留，MVP 不做。

### 后续版本再做

- 评论、点赞、收藏等家庭内轻互动。
- 原文件下载入口和短期授权下载。
- 实况照片动态播放。
- 批量下载。
- 最近添加视图。
- 拖拽上传。
- 拍照/录像直传。
- 分片上传和断点续传。
- 多码率视频转码。
- 手机后台上传或原生 App。
- 手机相册自动同步。
- 手机号短信验证或找回密码。
- 微信/Apple 登录。
- 更完整的上传任务中心。
- 批量修改拍摄时间。
- 家庭时区设置。

### 明确不做

- AI 整理、人脸识别、成长报告。
- 文件夹、标签、复杂搜索。
- 自动同步手机相册。
- 复杂角色权限。
- 宝宝模型和按宝宝筛选。
- 帖子、动态分组、上传批次作为浏览对象。
- 公开分享链接。
- 朋友圈式社交传播。
