## Why

MVP 已经打通家庭访问、上传、媒体处理和时间线浏览，但真实家庭试用前还缺少几个关键运营边界：管理员无法移除成员、不能保护最后一个管理员、错传媒体无法从时间线移除，处理失败也缺少明确恢复入口。

这些能力不适合后置太久。宝宝照片和视频属于敏感家庭数据，如果成员移除、媒体删除和失败恢复不扎实，产品即使能跑通主流程，也难以放心给家人长期试用。

## What Changes

- 增加家庭成员管理 API：管理员可以查看成员、修改成员称呼、移除成员。
- 增加最后一个 active `admin` 保护，避免家庭空间失去可管理者。
- 强化被移除成员的访问边界：被移除成员不能访问时间线、媒体详情、上传授权或上传任务。
- 增加媒体软删除能力：active `admin` 可以把错传或不想展示的媒体从家庭时间线和详情中移除，普通 `member` 不能删除媒体。
- 增加处理失败恢复能力：上传者或 active `admin` 可以对已上传原文件但预览生成失败的条目重新触发后端处理。
- 前端提供最小可用入口：管理员可执行成员移除、媒体删除和处理失败重试；普通成员只看到允许操作。
- 更新测试覆盖权限边界、最后管理员保护、软删除不可见和处理失败重试。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `family-access`: 增加成员列表、成员称呼更新、成员移除、最后管理员保护和 removed member 访问拒绝要求。
- `media-upload`: 增加处理失败重试要求，并保持重试不需要重新上传原文件。
- `timeline-browsing`: 增加管理员媒体软删除，以及已删除媒体在时间线和详情中不可见的要求。

## Impact

- Go API：新增成员管理、媒体删除、处理重试路由，并复用现有会话和 family membership 校验。
- Store 层：补充成员状态更新、最后管理员检查、媒体软删除、处理失败条目重新入队的数据操作；MemoryStore 和 MySQLStore 都需要保持一致。
- MySQL migration：如现有 schema 已包含软删除和成员状态字段，则优先复用；否则补最小 migration。
- Worker：处理失败重试应能让已有原文件重新进入待处理队列，不要求前端重新上传。
- React PWA：提供最小管理入口和错误恢复入口，但不做 Warm Editorial 大视觉迁移。
- 测试：覆盖管理员/普通成员/被移除成员的权限差异，以及时间线、详情和上传任务的边界行为。
