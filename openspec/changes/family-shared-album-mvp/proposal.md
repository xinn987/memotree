## Why

当前家庭成员分享宝宝精选照片、视频主要依赖 iCloud 共享相册、微信和各自手机相册，Android 体验和跨平台访问都不够稳定。第一阶段需要先建立一个轻量、私密、跨平台的家庭精选近况流闭环，让家庭成员可以安全上传并快速查看精选照片和视频。

## What Changes

- 引入家庭空间，家庭成员通过邀请加入同一个私密相册空间。
- 支持家庭成员上传精选照片、视频、实况照片和批量媒体文件。
- 上传后的媒体文件保存为私有原文件，并生成用于浏览和详情展示的缩略图、展示图和展示视频。
- 首页以时间线方式展示家庭媒体，按月份和日期组织内容。
- 支持查看单个照片或视频的详情，并播放视频预览。
- MVP 暂不提供原文件下载入口和下载 API，但保留原文件私有存储，为后续下载能力预留。
- 明确不在 MVP 中实现原文件下载、评论点赞收藏等家庭内轻互动、AI 整理、人脸识别、地图足迹、成长报告、文件夹管理、标签系统、自动同步手机相册和复杂权限模型。
- 评论、点赞和收藏属于后续可考虑的家庭内轻互动，不属于公开社交传播能力。

## Capabilities

### New Capabilities

- `family-access`: 家庭空间、成员加入、邀请和基础访问控制。
- `media-upload`: 照片、视频、批量上传、上传状态和媒体处理。
- `timeline-browsing`: 按时间线快速浏览家庭照片视频，并查看媒体详情。

### Modified Capabilities

无。

## Impact

- 需要新增 React + Vite 移动端 PWA，重点覆盖上传、时间线浏览和媒体详情。
- 需要新增 Go 后端 API，用于家庭成员访问控制、邀请、媒体元数据、上传会话和时间线查询。
- 需要使用 MySQL 保存用户、家庭、成员、邀请、媒体元数据和上传处理状态。
- 需要引入 Cloudflare R2 私有对象存储，用于保存原图、原视频、缩略图、展示图和展示视频。
- 需要引入媒体处理流程，用于提取基础元数据并生成适合浏览器展示的派生资源。
- 需要从第一版开始落实私密访问边界，避免直接暴露长期公开的媒体文件地址。
