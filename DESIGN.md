# Design

## Source Of Truth

MemoTree 的目标视觉系统来自 `design/` 目录中已经用 Impeccable 完成的设计探索稿，而不是当前 `web/` 目录里的旧 MVP 样式。

主要参考文件：

- `design/README.md`
- `design/tokens.css`
- `design/demo/assets/demo.css`
- `design/demo/timeline.html`
- `design/demo/media-detail.html`
- `design/demo/upload.html`
- `design/demo/auth.html`
- `design/demo/assets/shots/*.png`

当前 `web/src/styles.css` 仍然偏冷灰 SaaS 工具 UI，是后续迁移对象，不是目标设计基线。

## Visual Direction

视觉方向是“温暖纪实 / Warm Editorial”。MemoTree 应该像一本被家人认真整理过的宝宝成长相册：有纸面、胶片、手账和午后客厅的温度，但仍然是一个可靠的产品界面。

目标不是做怀旧装饰，也不是做营销页。设计服务于家庭浏览、上传、邀请和媒体详情这些产品任务。界面应当让照片自然成为主角，让家庭成员愿意慢慢看，同时让上传和管理动作清楚可执行。

## Color

颜色使用 `design/tokens.css` 中的 OKLCH tokens。

核心色彩角色：

- `--paper`: 燕麦暖白主背景。
- `--paper-raised`: 浅纸面，用于表单、轻面板和可操作区域。
- `--paper-recess`: 略深纸面，用于分隔、工具条和凹陷区域。
- `--ink`: 暖深棕主文字。
- `--ink-soft`: 次级文字。
- `--ink-faint`: 时间戳和提示文字，仍需保证可读性。
- `--accent`: 赭石砖红，用于主行动、选中态和强指示。
- `--caramel`: 焦糖色，用于月份标题、时间感和少量情绪强调。
- `--state-ok`, `--state-warn`, `--state-err`, `--state-info`: 收敛到暖域的语义状态色。

避免默认 SaaS 蓝、冷灰大面积背景、无语义的渐变装饰和过多单一色相变化。状态色可以清楚，但不能刺眼或破坏相册气质。

## Typography

字体系统：

- `Fraunces`: 现代衬线，用于品牌字、月份标题、页面标题、照片详情中的拍摄信息和手记式文案。
- `Inter`: 无衬线，用于 UI 正文、按钮、数据、输入框和元信息。
- 中文 fallback 使用 `Songti SC`, `Noto Serif SC`, `PingFang SC`, `Microsoft YaHei` 和系统字体。

产品 UI 使用固定字号为主，不做全页面流体字号。月份标题可以使用受控 `clamp()`，但最大值需要保持克制。展示字体的 letter spacing 不低于 `-0.04em`，当前目标值是 `-0.03em`。

## Layout And Information Architecture

### Timeline

时间线是主信息架构。首页围绕月份和日期组织照片，用“月度故事流”替代后台列表或统一卡片网格。

目标形态：

- 顶部是轻量品牌栏和家庭信息。
- 首屏问候使用衬线标题，例如“小满最近挺好的”。
- 月份分组使用大衬线标题，例如“六月 · 周岁”。
- 可选月份手记承载家庭记忆，例如“外婆特意从老家赶来”。
- 照片网格采用大小混排：特写大图、中等横图、竖图、小图组合。
- 照片直角或极小圆角，照片本身是主角。
- 元信息默认退后，hover 或 focus 时浮出。
- 上传入口使用右下 FAB 或等价的次级入口，保持可见但不抢浏览。

不要把时间线做成同尺寸卡片列表、后台素材库、文件夹视图或瀑布流工具页。

### Media Detail

媒体详情页是沉浸式照片查看。目标是“看一张照片”，不是“管理一条记录”。

目标形态：

- 背景使用暖深色，不用纯黑。
- 照片或视频居中聚焦，使用深沉但克制的展示阴影。
- 支持上一张、下一张切换，移动端和键盘交互应保留扩展空间。
- 侧栏元信息像相册背面的拍摄记录，使用衬线字。
- 移动端元信息可以折叠，避免压缩照片。
- 管理动作如下载原图、修改拍摄时间可以后置，不进入主视觉层级。

### Upload

上传是次要流程，但必须清楚可靠。文案优先家庭化。

目标形态：

- 页面标题使用“给家里加几张新的”这类自然说法。
- 文件选择热区清楚，移动端可直接从相册选择。
- 上传状态使用 chip 和细进度条表达。
- 状态文案使用“排队中”“传了 64%”“正在整理”“已整理好”“没传成”。
- 错误动作使用“再试一次”，停止动作使用“先不传了”。
- 最近上传可以保留，但不能变成任务中心主页面。

### Auth

登录页使用情绪照片板和极简表单，而不是居中 ghost-card。

目标形态：

- 桌面端左侧使用暖色家庭照片情绪板，右侧是登录表单。
- 移动端保留情绪板作为顶部 banner，不完全隐藏品牌温度。
- 首次加入通过家人邀请解释清楚。
- 表单视觉稳定、按钮清楚、提示文字可读。

## Components

组件词汇以 `design/demo/assets/demo.css` 为准：

- `app-bar`: 极简顶栏，左侧衬线品牌字，右侧家庭信息或成员头像。
- `btn`: `primary`, `ghost`, `text`, `danger`, `icon`。
- `field`, `input`, `select`: 表单基础组件。
- `chip`: 状态和轻量标签，允许 pill 形态。
- `photo`: 时间线照片单元，包含 hover/focus 元信息层。
- `month`: 月份分组标题和手记。
- `story-grid`: 大小混排照片网格。
- `toolbar`: 轻量筛选，不应显得像后台过滤器。
- `empty`: 教学式空状态。

卡片只在真正需要框住任务或表单时使用。页面 section 不应默认被包成浮动卡片。避免 card-inside-card。

## Spacing, Radius, Shadow

间距采用 4px 基准，但页面节奏需要有层级：章节级留白大于区块级，区块级大于元素级。

圆角保持克制：

- 小标签和照片使用 `4px` 或更小。
- 输入框、小按钮使用 `8px`。
- 任务面板等较大容器上限 `12px`。
- pill 只用于 chip、分段控件和 FAB。

阴影只用于明确的抬起或照片聚焦。避免 `1px border + 16px 以上大软阴影` 的 ghost-card 组合。普通面板优先使用边框或背景层级，不叠加装饰阴影。

## Motion

动效服务于反馈，不做编排式入场。

允许：

- 按钮 hover/active 的短过渡。
- 照片 hover 轻微 scale。
- 状态进度条宽度变化。
- 详情页切换时的轻量过渡。

所有动画必须有 `prefers-reduced-motion: reduce` 降级。内容不能依赖动画才可见。

## Copy Voice

默认中文。语气像家人之间共享相册，而不是后台系统。

推荐：

- “小满最近挺好的”
- “这是这周家里人拍下的瞬间。慢慢看。”
- “给家里加几张新的”
- “挑你想让家人看到的那些。”
- “这批正在传”
- “没传成”
- “再试一次”
- “先不传了”

避免：

- “任务状态”
- “处理失败”
- “上传批次”
- “媒体资产”
- “对象存储上传失败”
- “权限边界”

工程术语可以保留在开发文档和错误日志里，用户界面需要翻译成家庭成员能理解的语言。

## Accessibility

正文、输入框、按钮和状态提示需要满足 WCAG AA 对比度。老人是核心浏览用户，因此关键文本不能过小或过淡。

要求：

- 触控目标至少接近 40px 高，关键移动端操作更大。
- hover 信息也必须可通过 focus 或详情页访问。
- 状态不能只靠颜色表达，需要文字。
- 移动端不能出现按钮文字溢出、筛选控件挤压或图片遮挡操作。
- 详情页深色背景中的文字和按钮必须单独校验对比度。

## Migration Notes

从旧 `web/` 迁移到目标视觉系统时，优先顺序：

1. 接入 `design/tokens.css` 的核心 tokens 或在 `web/src/styles.css` 建立等价 tokens。
2. 改造时间线为月度故事流和大小混排照片网格。
3. 将上传从并列大面板降级为可见但次要的入口和任务页。
4. 改造媒体详情为沉浸式查看，并补齐上一张/下一张。
5. 改造登录页为情绪板 + 表单布局。
6. 全面替换后台式文案。

迁移时可以保留现有 React 数据流和 API 契约，但 className、CSS 和信息层级应向 `design/demo` 对齐。
